## 1. プロダクト概要

**SNS Growth OS** は、SNS運用の行動管理OS です。

投稿ツールではなく、**「毎日のSNS運用を継続できる状態を作る」** ことを目的としています。

### 解決する課題
- 毎日「何を投稿しよう？」と考えるのが負担
- SNS運用が三日坊主で終わる
- 投稿の振り返り・改善サイクルが回らない
- 競合分析の結果が自分のアクションにつながらない

### コアバリュー
- 朝アプリを開くだけで今日やることがわかる
- タスク量は直近の達成率に基づき自動調整される
- 1日サボっても翌日リカバリーできる設計
- AIは補助的に使い、コスト月3,000円以内に抑制

## 3. 画面構成と機能一覧

### 画面遷移図

```
ログイン ─→ ダッシュボード（ホーム）
                  │
                  ├── 今日の運用（タスク管理）
                  ├── コンテンツ作成（AI文案生成）
                  ├── 投稿ログ（記録・メトリクス）
                  ├── 分析（パフォーマンス）
                  ├── 参考投稿（競合分析）
                  ├── AI相談（チャット）
                  └── 設定
```

### 3-1. ダッシュボード（/dashboard）

**役割**: ユーザーが毎朝最初に見る画面。今日の全体像を一目で把握。

| 要素 | 内容 |
|------|------|
| 日付表示 | 今日の日付 + 残タスク数のサマリー |
| 達成率カード | 今日のタスク完了率をプログレスバーで表示 |
| 完了タスクカード | 完了数 / 総数 |
| 優先媒体カード | 設定した優先プラットフォーム |
| 継続状況カード | 直近7日間の達成率を色分けブロックで表示 |
| 今日のタスク一覧 | タスクカード形式。クリックで完了/未完了を切替 |
| クイックアクション | 文案作成・今日の運用・参考投稿分析・分析への導線 |

**レスポンシブ対応**:
- モバイル: KPIカード2列、タスク一覧は縦スクロール、下部タブナビ
- デスクトップ: KPIカード4列、タスク一覧(60%) + クイックアクション(40%)の2カラム、左サイドバー

---

### 3-2. 今日の運用（/today）

**役割**: 今日のタスクを時刻順に管理・実行する画面。

| 機能 | 詳細 |
|------|------|
| 進捗バー | 完了数/総数、完了率、残り所要時間を表示 |
| タスク一覧 | 時刻順に表示。各タスクに媒体バッジ・時刻・所要時間・ステータス |
| 完了操作 | チェックボタンで完了/未完了を切替 |
| 延期操作 | カレンダーアイコンで翌日に繰越 |
| スキップ操作 | スキップボタンでスキップ扱い |
| タスク追加 | ユーザーが任意のタスクを手動追加（媒体・タイプ選択可） |
| 日次レポート生成 | AIが今日の振り返り・良かった点・改善点・明日の優先事項を生成 |
| 文案作成への導線 | 投稿作成系タスクからペンアイコンでコンテンツ作成画面へ遷移 |

---

### 3-3. コンテンツ作成（/create）

**役割**: AI を使ってSNS投稿の文案を生成する画面。4タブ構成。

| タブ | 生成内容 |
|------|----------|
| X投稿 | 140文字想定のテキスト投稿。フック→本文→CTA構成 |
| Instagram | キャプション。フック+本文+CTA+ハッシュタグ |
| note記事 | 1000-2000文字の記事下書き。タイトル+見出し+本文+まとめ |
| 動画プロンプト | 外部動画ツール用の台本（シーン構成・ナレーション・テロップ・尺） |

| 入力項目 | 説明 |
|----------|------|
| テーマ（必須） | 投稿の主題 |
| トーン | プロフェッショナル / カジュアル / フレンドリー 等6種 |
| CTA | フォロー / いいね / コメント / シェア / リンク / 保存 |
| 追加コンテキスト | ターゲット層やキーワード等の自由入力 |

| 出力操作 | 説明 |
|----------|------|
| コピー | クリップボードにコピー |
| 下書き保存 | DBに下書きとして保存 |

動画プロンプトタブでは Higgsfield への外部リンクを表示。

### 3-4. 投稿ログ（/posts）

**役割**: 実際に投稿した内容を記録し、結果を追跡する画面。

| 機能 | 詳細 |
|------|------|
| 投稿記録 | 媒体・テーマ・フック・本文・CTA・投稿タイプ・URLを入力 |
| 投稿一覧 | カード形式で時系列表示。媒体バッジ・日付・メトリクスを表示 |
| 結果入力 | 各投稿にメトリクスを記録（表示回数・いいね・コメント・シェア・保存） |

※ MVP段階ではSNS APIとの自動連携は行わず、手動入力を基本とする

### 3-5. 分析（/analysis）

**役割**: 投稿パフォーマンスとタスク達成率を俯瞰する画面。

| 要素 | 内容 |
|------|------|
| KPIカード（4列） | 総投稿数 / 平均いいね / 平均表示回数 / 平均タスク達成率 |
| 達成率推移グラフ | 日別の達成率を棒グラフで表示（色分け: 緑80%以上 / 黄50%以上 / 赤50%未満） |
| 投稿パフォーマンステーブル | 日付・媒体・テーマ・表示回数・いいね・コメント・シェアを一覧表示 |

### 3-6. 参考投稿 / 競合分析（/trends）

**役割**: 競合や参考アカウントの投稿を保存し、AIで構造分析する画面。

| 機能 | 詳細 |
|------|------|
| ソース登録 | 競合アカウントや参考URLを名前・URL・媒体・タイプで登録 |
| 参考投稿追加 | 投稿タイトル・フック・全文・URL・メモを手動入力 |
| AI分析 | 保存した投稿をAIが分析。以下を抽出: |
|  | - フックパターン（なぜ冒頭が効果的か） |
|  | - 構成パターン（全体の構造） |
|  | - CTAパターン（行動喚起のやり方） |
|  | - トーン分析 |
|  | - なぜ伸びたかの推測 |
|  | - 自分用アクション提案 |

**最終目的**: 分析結果を「今日の自分用アクション」に変換できること

**想定される相談例**:

## 5. タスク自動生成ロジック（AIではなくルールベース）

**このアプリの最重要機能。** ユーザーが毎日何をすべきか自動で提示する。

### 負荷調整ルール

| 条件 | 動作 |
|------|------|
| 直近3日の完了率 >= 80% | 好調モード: タスク量を増やす（8件程度） |
| 直近3日の完了率 50-80% | 通常モード: 標準タスク量（6件程度） |
| 2日連続で完了率 < 50% | リカバリーモード: 最低限のタスクのみ（4件） |

### 媒体別スケジュール

| 媒体 | 頻度 | 理由 |
|------|------|------|
| X | 毎日 | 軽量（15分/件）で高頻度が効果的 |
| Instagram | 通常・好調モード時 | 中負荷（20分/件） |
| note | 週2回（水・土に構成、木・日に執筆） | 重負荷（30-45分/件） |
| 動画プロンプト | 週2回（火・金） | 外部ツール連携のため準備のみ |

### リカバリーモードの例

```
通常日（6件）:              リカバリー日（4件）:
- X投稿案作成               - X投稿案作成
- X投稿を公開               - X投稿を公開
- Xエンゲージメント活動      - 参考投稿を分析
- Instagramキャプション作成  - 今日の振り返り
- 参考投稿を分析
- 今日の振り返り
```

## 6. AI利用方針

### AIを使う場面（月3,000円以内）

| 機能 | 頻度 | 用途 |
|------|------|------|
| 日次レポート | 1日1回 | 達成の振り返り・改善提案 |
| 文案生成 | 都度（ユーザー操作時） | X/Instagram/note/動画プロンプト |
| AI相談 | 都度（ユーザー操作時） | SNS運用全般の質問 |
| 競合分析 | 都度（ユーザー操作時） | 参考投稿の構造分析 |

### AIを使わない場面（システムロジック）

- タスクの自動生成・負荷調整
- タスク繰越処理
- 連続継続日数の計算
- 実行率・完了率の集計
- 投稿数の最低ライン維持判定

### コスト制御設計

- AI Provider抽象化層を実装済み（mock / OpenAI 切替可能）
- モデル名・温度・最大トークン数を環境変数で切替可能
- 使用量ログテーブル（ai_usage_logs）で日次・月次コスト追跡
- チャットは直近10件のみ文脈として送信（全履歴を投げない）

## 8. データベース設計（13テーブル）

| テーブル | 用途 | 主要カラム |
|----------|------|------------|
| profiles | ユーザー情報 | display_name |
| workspaces | ワークスペース | goal_followers, primary_platform |
| social_accounts | SNSアカウント | platform, account_name, current_followers |
| daily_tasks | 日次タスク | task_date, platform, status, scheduled_time |
| content_drafts | コンテンツ下書き | platform, content_type, body, tone |
| posts | 投稿ログ | platform, theme, hook, body, cta |
| post_metrics | 投稿メトリクス | views, likes, comments, shares, saves |
| trend_sources | 競合/参考ソース | source_type, platform, source_name |
| trend_posts | 参考投稿 | title, hook, body_summary, raw_text |
| trend_analyses | 投稿分析結果 | analysis_summary, hook_pattern, suggested_action |
| daily_reports | 日次レポート | completion_rate, summary, ai_insight |
| ai_conversations | AIチャット履歴 | role, content, message_group_id |
| ai_usage_logs | AI利用量ログ | provider, model, input_tokens, estimated_cost |

全テーブルに Row Level Security を適用（ユーザーは自分のデータのみアクセス可能）。
